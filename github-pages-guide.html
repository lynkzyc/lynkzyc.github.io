1 注册 GitHub 账号

一切开始的地方

1

创建你的 GitHub 账号

2 分钟 · 免费 · 需要邮箱
▾

打开 github.com，点击右上角 Sign up。

  • 输入你的邮箱（推荐常用邮箱，QQ邮箱也行）
  • 设置密码和用户名（用户名会出现在你的网站地址里，想好再填）
  • 完成验证码，点击创建账号
  • 收到验证邮件后，在邮件里点确认
⚠️
用户名很重要！你的网站地址会是 你的用户名.github.io，选一个你愿意展示给别人看的名字。
2 创建一个仓库

仓库 = 存放网站文件的地方

2

新建 GitHub 仓库

1 分钟 · 仓库名有讲究
▾

登录后，点击右上角 + → New repository。

仓库名有两种选择：

类型仓库名网站地址
个人/组织主页 用户名.github.io https://用户名.github.io
项目页面 任意名字 https://用户名.github.io/仓库名
  • Public 必须选公开（免费账户公开仓库才能用 Pages）
  • 勾选 Add a README file（可选，方便后面操作）
  • 点击 Create repository
🔑
如果你是新手，建议直接用 用户名.github.io 作为仓库名。一个账号只能有一个这种仓库，它会自动开启 Pages 功能。
3 上传你的网页文件

把 HTML 文件放进仓库

3

3 种上传方式，任选一种

按自己水平选最简单的方式
▾

方式一：网页直接上传（推荐小白）

  • 进入仓库页面，点击 Add file → Upload files
  • 把你的 index.html 和其他文件拖进去
  • 下面写一句描述（比如"第一次上传"），点 Commit changes

方式二：在网页上直接创建（超快）

  • 进入仓库，点击 Add file → Create new file
  • 文件名填 index.html
  • 粘贴你的 HTML 代码，点 Commit

方式三：Git 命令行（进阶）

$ git clone https://github.com/用户名/用户名.github.io.git
$ cd 用户名.github.io
$ echo '<h1>Hello World</h1>' > index.html
$ git add . && git commit -m "first deploy"
$ git push
4 开启 GitHub Pages

让网站真正上线

4

设置 Pages 并等待部署

2 分钟 · 最重要的一步
▾

进入仓库页面 → 顶部点 Settings → 左侧菜单点 Pages。

  • Source：选择 Deploy from a branch
  • Branch：选择 main（或 master），目录选 /(root)
  • 点击 Save

保存后，GitHub 会自动开始部署。稍等 1-2 分钟，刷新页面。

页面顶部会出现：✅ Your site is live at https://用户名.github.io

⏱️
如果仓库名是 用户名.github.io，推代码后会自动部署，不需要手动去 Settings 开启。但手动确认一下更放心。
5 访问你的网站

恭喜，你的网站上线了！

5

打开浏览器，输入网址

看到页面就是成功了 🎉
▾

在浏览器地址栏输入：https://你的用户名.github.io，回车。

能看到你的网页，就算成功了！把这个链接发给朋友，他们在自己手机上也能打开。

# 一个典型的 GitHub Pages 地址
https://zhangsan.github.io
https://zhangsan.github.io/my-project

没看到页面怎么办？

  • 确认仓库是 Public（Settings → 拉到最下面 Danger Zone 可以看到）
  • 确认仓库根目录有 index.html 文件
  • 等几分钟，首次部署需要一点时间
  • 去 Settings → Pages 看有没有红色报错
  • 清除浏览器缓存再试（Ctrl+Shift+R）
6 更新网站 & 进阶玩法

网站上线只是开始

6

日常更新 & 更多可能

改文件 → Commit → 自动生效
▾

如何更新网站？改完文件 → 上传/Commit → 等 30 秒 → 刷新网站，完事。

进阶玩法：

玩法怎么做
绑定自己的域名Settings → Pages → Custom domain，然后在域名商加一条 CNAME 记录
用 Jekyll 主题Settings → Pages → Theme Chooser，一键换模板
用静态网站生成器Hugo / Hexo / VuePress / Next.js 导出 → 推到仓库即上线
GitHub Actions 自动构建写一个 .github/workflows 文件，推代码自动编译部署
多页面网站在仓库里创建 about.html、contact.html 等，自动生成对应路径
📚
官方文档：docs.github.com/pages — 遇到问题先查这里。
