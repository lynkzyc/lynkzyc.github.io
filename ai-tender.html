AI招投标辅助系统 — 架构总览

统一数据底座 · 五段流水线 · 人工门控 · 结果反馈闭环

底

零、知识库管理（系统数据底座）

全系统的数据地基 · 所有 Agent 共享读写

知识库是整个系统的底座。所有 Agent 的输入依赖它，输出回写它。没有它，分析和生成都是"无米之炊"。

公司资质管理
营业执照、行业资质证书、专利等
历史业绩库
过往中标项目、合同、验收报告及客户评价
人员证书管理
技术人员资质、职称证书、社保证明
技术方案库
标准化技术方案模板与可复用模块组件
历史投标文件库
归档管理、版本追溯与复用检索
统一标书数据对象
贯穿全流程的标准数据结构，各 Agent 共享读写
标书基本信息 分类标签 解析结果 评估结论 投标决策 生成状态
1

一、标书抓取 Agent

数据获取入口 · 抓取标书信息与相关文档
  • 模拟登录
  • 抓取当天的标书信息，相关文档的下载
  • 生成获取标书的标准表格数据写入 → 统一标书数据对象
2

二、标书分类 Agent

AI 识别项目类型 · 关注 / 不关注领域分类
  • 根据抓取的标书信息，AI 识别该招标项目的类型
  • 分类（关注领域和不关注领域）写入 → 分类标签字段
  • 生成标书分类汇报的标准表格数据更新 → 统一标书数据对象
! 人工审核门控 · 确认进入深度分析的标书清单
3

三、标书解析与分析报告 Agent

文本提取 · 结构化解析 · 评估打分 · 生成报告
  • 获取关注领域的标书读取 ← 统一标书数据对象
  • 提取文本
    将招标网页信息、PDF、Word、Excel 等相关数据进行识别和提取
  • 结构化解析回填 → 统一标书数据对象
    将相关数据进行过滤和解析，按字段回填
  • 验证交付读取 ← 知识库底座
    综合知识库底座数据，对比标书要求，给出评估意见和打分
    • 项目概述（元数据 + 评分构成 + 关键时间节点）
    • 评分策略（每项评分的目标得分 + 具体策略）
    • 技术要求分析
    • 商务要求分析
    • 风险清单
    • 资质准备清单
    • 人员配置与证书要求
  • 生成 Word 分析报告写入 → 评估结果字段
    分析结论、评分总结、推荐 / 不推荐理由
! 投标决策门控 · 根据分析报告，人工决定是否投标、确定策略方向
4

四、投标自动生成 Agent

需求分析 · 框架生成 · 内容填充 · 检查优化
以 Agent 3 分析报告和投标决策门控结论为输入，严格基于分析结果驱动生成
  • 需求分析
    • 获取招标文件核心内容
    • 基于分析报告匹配投标方核心优势与业绩
    • 确认技术标 / 商务标的分值占比
  • 框架生成
    • 按评分标准生成文档结构
    • 标注每个章节对应的得分项
    • 建议各章节的篇幅和重点
  • 内容填充读取 ← 知识库底座（技术方案库 + 资质数据）
    • 技术方案：针对需求设计解决方案
    • 项目实施：制定进度计划和保障措施
    • 商务响应：资质证明和偏离表
  • 检查优化
    • 对照评分标准逐项检查覆盖度
    • 排查废标风险（格式 / 签章 / 资质）
    • 优化措辞提升专业度
    • 去 AI 味检查
5

五、投标结果反馈

记录结果 · 分析得失 · 回流知识库
  • 记录投标结果
    中标 / 落标 / 流标
  • 汇总评审打分明细与扣分项分析
  • 结果数据回流至知识库底座写入 → 知识库底座
    • 中标项目：提取中标策略要素，丰富技术方案库和历史业绩库
    • 落标项目：分析失分原因，优化后续标书分类策略和投标策略
    • 归档投标文件全文与评审反馈到历史投标文件库
